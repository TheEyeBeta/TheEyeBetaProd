# CI/CD Documentation

## Overview

Five GitHub Actions workflows under `.github/workflows/`. No Python version matrix — `3.12` is
pinned everywhere; an earlier draft of this doc described a 3.11/3.12/3.13 matrix that was never
implemented.

## `ci.yml` — runs on every push/PR

Job graph:

```
lint ──┬── py-test ── integration-tests ── smoke-staging ──┐
       ├── cpp-build ──────────────────────────────────────┤
       ├── sbom ───────────────────────────────────────────┤
       └── docs ───────────────────────────────────────────┴── all-ok
```

`lint`, `cpp-build`, `sbom`, and `docs` all run in parallel once `lint` itself passes; `py-test` →
`integration-tests` → `smoke-staging` is a linear chain after `lint`. `all-ok` requires all seven
jobs to succeed and is the single required status check for branch protection.

| Job | What it does |
|-----|---------------|
| `lint` | `ruff check`, `ruff format --check`, `mypy --strict` (libs/), `clang-format --dry-run`, `sqlfluff lint` |
| `py-test` | `pytest -m "not integration and not smoke"` |
| `integration-tests` | `pytest -m integration` against testcontainers |
| `smoke-staging` | Brings up `docker compose`, runs `pytest -m smoke` against it |
| `cpp-build` | Matrix build (ubuntu-22.04, macos-14) — CMake + Conan + ctest |
| `sbom` | Generates an SBOM via `anchore/sbom-action` (syft) |
| `docs` | `make docs-api-check` — fails if `docs/api/*.openapi.json` is stale (fix with `make docs-api`) |

## `paper-smoke.yml` — separate from CI smoke

Nightly, weekday cron, **self-hosted runner** (`[self-hosted, production]`). Hits *live*
paper-trading endpoints (`/health`, `/v1/macro/*`, etc.) on the production host. This is distinct
from the `smoke-staging` job above, which runs against ephemeral compose infra on a GitHub-hosted
runner — don't conflate the two when debugging a "smoke test" failure.

## `release.yml`

Triggered by `v*.*.*` tags: builds container images for the deployed services, publishes the `tb`
CLI, and cuts a GitHub Release with notes generated by `git-cliff` (`cliff.toml`).

## `deploy.yml`

Triggered by pushes to `main` (gated via the `production` GitHub Environment): SSHes to the Mac
mini over Tailscale, runs `docker compose up`, then `tb status` to confirm health.

## `bench.yml`

Performance/benchmark workflow for the C++ compute layer (`cpp/bench/`).

## Branch Strategy

- `main`: protected, requires `all-ok` green + 1 approval
- `feat/*`, `fix/*`, `chore/*`, `refactor/*`, `ci/*`: feature branches, CI runs on PR
- Merge strategy: squash merge to `main`

## Secrets Required in GitHub

| Secret Name | Used in | Purpose |
|-------------|---------|---------|
| `CODECOV_TOKEN` | `ci.yml` | Coverage upload |
| `SSH_HOST`, `SSH_USER`, `SSH_PRIVATE_KEY` | `deploy.yml` | SSH to the Mac mini |
| `TS_OAUTH_CLIENT_ID`, `TS_OAUTH_SECRET` | `deploy.yml` | Tailscale OAuth for the deploy runner |
| `GHCR_TOKEN` | `release.yml` | Push images to GHCR |
| `ALPACA_API_KEY_PAPER`, `ALPACA_API_SECRET_PAPER` | `paper-smoke.yml` | Live paper-trading credentials |
| `DATABASE_URL`, `NATS_URL`, `REDIS_URL` | `paper-smoke.yml` | Connect to production infra for the nightly smoke check |

## Cache Strategy

- `uv` dependency cache keyed on `pyproject.toml` hash
- Docker layer cache via `actions/cache` with `type=gha`
- C++ build cache via `ccache`

## Failure Policy

- Any failed job in `ci.yml` blocks merge via the `all-ok` required check.
- `paper-smoke.yml` failures are advisory/alerting (nightly, not a merge gate).
- Flaky tests must be fixed within one sprint or quarantined with `@pytest.mark.skip` plus a
  linked issue (`.claude/rules/01-code-style.md` — no bare TODOs).
